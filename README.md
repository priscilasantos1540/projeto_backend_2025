# Sistema de Gerenciamento de Eventos IFNMG (ainda sem nome)
Projeto realizado como instrumento avaliativo para a disciplina de Desenvolvimento Web Back-End e em colaboração com todo o restante da turma.

# Instruções para a turma:
Para cada grupo começar a desenvolver suas respectivas partes, o líder do grupo deverá **criar um fork** (uma cópia independente deste repositório, mas na conta do criador) do repositório e conceder permissões de commit no fork criado aos demais membros. Os commits do grupo serão realizados *no fork do líder* e, apenas ao final da parte em questão, será realizado um pull request para o repositório central (este, também chamado de *upstream* nesse contexto).

Caso algum dos conceitos citados não seja do conhecimento de algum membro, é **fundamental** que os conceitos básicos de Git e GitHub sejam estudados/revisados antes de realizar contribuições.

# Atenção!!!
Para os grupos que realizarão trabalhos em paralelo, é importante que se atentem ao estado atual da upstream. Caso novos commits tenham sido feitos ou pull requests de outros grupos aceitos, é necessário que os forks dos grupos que ainda estão trabalhando sejam sincronizados antes de fazer pull requests.
